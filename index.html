C++ Study Guide

1. C++ Basics

  • Algorithm
    • A step-by-step procedure or formula for solving a problem
  • Types of errors:
    • Syntax: Errors in the structure of the code
    • Runtime: Errors that occur during program execution
    • Logic: Errors in the program's logic or design
  • ASCII Table
    • A character encoding standard for electronic communication
    • Used to represent text in computers and other devices
  • Language types:
    • Machine language: Binary code directly executed by the computer
    • Assembly: Low-level programming language with a strong correspondence to machine code instructions
    • High-Level: More abstract languages that are easier for humans to read and write (e.g., C++, Python)
  • Software life cycle steps
    • Typically includes: Analyze the problem, Design, Implementation, Testing, Deployment, and Maintenance
  • Variable declaration, initialization, and assignment
    • Declaration: Specifying a variable's type (e.g., int x;)
    • Initialization: Giving a variable its first value (e.g., int x = 5;)
    • Assignment: Changing a variable's value (e.g., x = 10;)
  • Valid variable naming conventions
    • Can include letters, digits, and underscores
    • Must start with a letter or underscore
    • Case-sensitive
    • Cannot use C++ keywords
  • Purpose and usage of #include
    • Preprocessor directive that includes the contents of another file in the current file
    • Used to include header files containing function declarations and definitions
  • Common operators:
    • =: Assignment operator
    • +=, -=, *=, /=, %=: Compound assignment operators
    • ++, --: Increment and decrement operators
    • +, -, *, /, %: Arithmetic operators
    • ==, !=, <, >, <=, >=: Comparison operators
    • &&, ||, !: Logical operators
    • &, |, ^, ~, <<, >>: Bitwise operators
    • ?:: Ternary conditional operator
    • ,: Comma operator
    • sizeof: Size-of operator
    • &, *: Address-of and dereference operators
  • Difference between = and ==
    • = is for assignment
    • == is for comparison (equality check)
  • Type casting in C++
    • Converting one data type to another
    • Example: static_cast<int>(doubleValue)

2. Linux/Git Commands

  • cp: Copy files and directories
  • ls: List directory contents
  • rm: Remove files or directories
  • mv: Move or rename files and directories
  • touch: Create empty files or update file timestamps
  • mkdir: Create directories
  • cd: Change directory
  • Git commands:
    • clone: Create a copy of a remote repository
    • status: Show the working tree status
    • commit: Record changes to the repository
    • add: Add file contents to the index
    • push: Update remote refs along with associated objects
    • pull: Fetch from and integrate with another repository or a local branch

3. Flow Control

  • Modulus operator: %
    • Returns the remainder of a division operation
  • Logical AND: &&
    • Returns true if both operands are true
  • Logical OR: ||
    • Returns true if at least one operand is true
  • Equality operator: ==
    • Checks if two values are equal
  • Short circuit evaluation
    • Stops evaluating a logical expression as soon as the result is determined
  • For loops vs. While loops: Usage scenarios
    • For loops: Used when the number of iterations is known
    • While loops: Used when the number of iterations is unknown
  • Loop control variables and scope
    • Variables declared in the loop have local scope
    • Variables declared outside the loop have broader scope
  • Types of loops: for, while, do-while, infinite loops
    • For: Executes a specified number of times
    • While: Executes as long as a condition is true
    • Do-while: Executes at least once, then continues while a condition is true
    • Infinite: Loops that don't have a termination condition

4. Variable Scope and Lifecycle

  • Local vs. global variables
    • Local: Limited to the block where they are declared
    • Global: Accessible throughout the entire program
  • Variable initialization and default values
    • Uninitialized variables may contain garbage values
    • Best practice: Always initialize variables
  • Lifecycle of variables in different contexts
    • Variables in loops: Created and destroyed with each iteration (if declared inside)
    • Function parameters: Exist for the duration of the function call

5. Best Practices and Common Pitfalls

  • Proper variable naming and conventions
    • Use descriptive names
    • Follow consistent naming conventions (e.g., camelCase or snake_case)
  • Avoiding unintended behavior in loops
    • Be careful with loop counters and termination conditions
    • Watch out for off-by-one errors
  • Proper use of assignment vs. equality operators
    • Use = for assignment, == for comparison
    • Common mistake: Using = in conditional statements
  • Understanding variable scope and its implications
    • Be aware of where variables are accessible
    • Avoid naming conflicts between local and global variables
